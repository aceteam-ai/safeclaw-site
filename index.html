SafeClaw

Safety proxy for AI agents. Runs in a Docker container that can't access your files, email, or credentials. Every LLM call gets tracked, dangerous actions get blocked, every verdict is cryptographically signed. Open source.

Get Started

Run this in your terminal. Requires Docker.

mkdir -p ~/safeclaw
docker run -p 8899:8899 -v ~/safeclaw:/workspace ghcr.io/aceteam-ai/aep-proxy

Then open http://localhost:8899/aep/ to configure your API key and see the dashboard. The container can only access ~/safeclaw — nothing else on your machine.

Workshop participants: If your instructor provided a proxy URL, you don't need to install anything. Just set OPENAI_BASE_URL to the URL they gave you and run your agent.

What It Does

  • Tracks cost — every call, every token, every model. Real-time dashboard.
  • Blocks threats — port scans, exploit code, credential access. Blocked before reaching the LLM. $0 cost.
  • Detects PII — SSN, credit card, email in responses. Blocked before your agent sees it.
  • Signs verdicts — Ed25519 signed, Merkle chained. Cryptographic proof that safety checks ran.
  • Enforces your policy — one YAML file per company. Healthcare, finance, startup — different rules, same engine.

Works with OpenClaw, NemoClaw, NanoClaw, CrewAI, LangChain, Claude Code, or any OpenAI-compatible agent.

What is AEP?

AEP (Agentic Execution Protocol) is the open protocol behind SafeClaw. It defines how agent calls are tracked, how safety is enforced, and how verdicts are signed. The protocol is Apache 2.0 — anyone can implement it. SafeClaw is the reference implementation.

Four pillars: cost tracking, safety enforcement, provenance (where data came from), and governance (who accessed what, with what consent).

Alternative Install (pip)

If you prefer not to use Docker (developer mode — runs on your host):

pip install aceteam-aep[all]
aceteam-aep proxy --port 8899

Note: pip mode runs directly on your machine — the proxy has access to your filesystem. Docker mode is recommended for isolation.
